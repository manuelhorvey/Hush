# Hush v0.1: Product Foundation Document — Part 6: Design System & UI/UX

---

## 90. Design Philosophy
Hush is designed for **presence, not accumulation**. The interface creates a private, intimate environment, avoiding the clutter and retention-based habits of traditional messaging apps.

## 91. The Emotional Journey
Curiosity $\rightarrow$ Trust $\rightarrow$ Comfort $\rightarrow$ Connection $\rightarrow$ **Completion** $\rightarrow$ Peace.

## 92. Brand Personality
Calm, human, premium, and trustworthy. The brand avoids "military" or "hacker" aesthetics, opting for peaceful, elegant privacy.

## 93. Visual Direction
*   **Theme:** Dark mode by default (Charcoal backgrounds #111111, Surface #1A1A1A, Text #F5F5F5).
*   **Accents:** Soft indigo/blue for trust; neutral/fading tones for destruction.
*   **Typography:** Modern, clean, and human (Inter or SF Pro).

## 94. Information Architecture
Minimalist structure: **Hush (Active Conversations)**, **Contacts**, and **Settings**.

## 95. UI/UX Rules
*   **Destruction Experience:** Peaceful dissolution (ink fading or sand falling). Never use red/error colors for completion.
*   **Empty States:** Reflect the philosophy: "Nothing here because nothing needs to remain."
*   **Notifications:** Must never leak content; use generic "New Hush conversation" alerts.
*   **Lifecycle Settings:** Simple, non-technical choices (e.g., "After both reply").

## 96. UX Design Principles
1.  **Encourage Disappearance:** If a feature encourages keeping things, remove it.
2.  **Trust-Centric:** Every error message and loading state must reinforce security and reliability.
3.  **Human Focus:** The interface serves only to help a conversation happen.

## 97. The Signature Moment
The "Destruction Receipt"—the screen showing "Conversation destroyed" and "This moment has ended"—is the core magical interaction.

## 98. Success Criteria
Users immediately understand the model, feel calm, trust the privacy, and don't miss the traditional "archive" features.
